## 二、遇到的问题

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | 注册 API 返回 `Attempt to export non-extractable HMAC key` | `deriveKey()` 第四个参数 `extractable` 设为 `false`，`hashPassword` 中 `exportKey('raw')` 失败 | 改为 `true`，因为必须导出 raw hash 用于存储和验证 |
| 2 | 登出 API 返回 `[object Response]` | `handleLogout` 中 `new Response(json(...))` — `json()` 本身返回 `Response`，外层再包一层导致 `toString()` | 改用 `JSON.stringify()` + 单层 `new Response()` |

## 三、关键决策

| # | 决策点 | 选项A | 选项B | 选择 | 理由 |
|---|--------|-------|-------|------|------|
| 1 | 数据存储 | D1 (Cloudflare 托管 SQLite) | 纯 JSON 文件 | D1 | 需求文档原要求 JSON，但 D1 免费额度远超需求、自带迁移、与 Pages Functions 集成好；JSON 文件并发写危险 |
| 2 | 密码哈希 | PBKDF2 (Web Crypto API) | bcrypt / argon2 (需 WASM) | PBKDF2 | Workers 环境原生支持 Web Crypto，无额外依赖，100k 迭代足够 |
| 3 | JWT 实现 | 手动 HMAC-SHA256 JWT | jose 库 | 手动 | 零依赖，避免 Workers 打包体积膨胀，HS256 逻辑简单可靠 |
| 4 | 邮箱验证 | 注册即自动验证 | 发送验证邮件 | 自动验证 | 小体量 wiki，暂无邮件服务；保留 `markEmailVerified` + 验证端点，后续可接入 MailChannels |
